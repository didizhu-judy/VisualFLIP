VisualFLIP Do Answers Follow Evidence?答案会跟随证据吗？

Didi Zhu, Changrui Chen, Stefanos Zafeiriou, Jiankang Deng

Imperial College London

VisualFLIP tests whether multimodal predictions actually update when task-critical visual evidence changes. Each item keeps the question fixed, flips only the image evidence, and asks whether the model moves with the gold answer instead of repeating itself. VisualFLIP 测试多模态模型的预测是否真的会随任务关键视觉证据更新。每个样本保持问题不变，只翻转图像证据，观察模型是否跟随标准答案变化，而不是固守原答案。

01 / Overview概览

A correct answer is not always a grounded one.答对，未必意味着有据。

Modern multimodal LLMs answer many visual-reasoning questions correctly. But single-image accuracy can hide answers that are not driven by the visual evidence — driven instead by language priors, memorized regularities, or whatever a confident reasoning chain happens to settle on. VisualFLIP turns this into a behavioral test. 现代多模态大模型能答对很多视觉推理题。但单图准确率会掩盖那些并非依赖视觉证据的答案——它们可能源自语言先验、记忆的规律，或一段听上去自洽的推理链恰好落到的位置。VisualFLIP 把这件事变成一个行为测试。

One pair = one question, two images. The second image makes a minimal task-critical edit that deterministically flips the gold answer. A prediction sensitive to the evidence should change between the two; a prediction that repeats the same answer despite the gold flipping collapses. The benchmark is behavioural — it measures whether predictions move with the evidence, not the internal mechanism behind any individual answer. 一对样本 = 一个问题，两张图。第二张图做一个对任务关键的最小化编辑，使标准答案确定性翻转。对证据敏感的预测应当随之改变；若两张图给出相同答案而标准答案已翻转，则发生坍塌。这是一个行为基准——它衡量预测是否随证据更新，并不主张任何关于内部机制的结论。

03 / Metrics评测指标

Pair accuracy and Collapse Rate.配对准确率与坍塌率。

Two pair-level numbers carry the report; a third is reported on the controls. 我们用两个配对级别的指标作为主报告，再在对照样本上报告第三个。

Accp — pair accuracy配对准确率

The model must solve both sides of a pair to score. Direction-free, paired analog of single-image accuracy. Higher is better. 必须两侧都答对才计分；配对设置下方向无关的准确率类比。越高越好。

CR — Collapse Rate坍塌率

Of the pairs the model solves on at least one side, the fraction where it gives the same non-empty answer to both images despite the gold flipping. Low CR is consistent with predictions that move with the task-critical evidence; high CR shows the prediction failed to update. Lower is better. 在模型至少一侧答对的样本中，两侧给出相同非空答案、但标准答案已翻转的比例。CR 低意味着预测随任务关键证据而变化；CR 高则表示预测未更新。越低越好。

S / F — control preservation / false-flip对照保持率 / 误翻率

On the 140 pairs that ship an irrelevant_image arm (templates hard_dense_count, attr_dense_color_count, layer_order): S = the prediction stayed when the irrelevant edit didn't change the gold (higher is better), F = the prediction changed anyway (lower is better). 在带有 irrelevant_image 对照臂的 140 对样本上（模板：hard_dense_count、attr_dense_color_count、layer_order）：S=无关编辑下答案保持（越高越好），F=无关编辑下答案改变（越低越好）。

Evaluation protocol (independent mode)评测协议（独立模式）

For every pair, the same question is asked on the original and edited images as two separate completions — not a multi-turn conversation. The prompt forces an answer tag, and the extractor falls back through <answer> → \boxed{} → trailing letter (A–E) or integer. 每对样本，同一问题分别在原图与编辑图上以两次独立的请求询问，而非多轮对话。提示强制要求答案标签，抽取顺序为 <answer> → \boxed{} → 末行尾部字母（A–E）或整数。

06 / Findings主要发现

Three patterns across the 24 models.24 个模型之间的三条规律。

  1. Accp does not determine CR.Accp 决定不了 CR。 Even frontier systems collapse on 5–13% of solvable pairs. High pair accuracy does not imply low collapse. 即便是前沿系统，在它们能解的样本中仍有 5–13% 出现坍塌。配对准确率高并不蕴含坍塌率低。
  2. Tool-augmented models in this evaluation remain vulnerable.本评测中的工具增强模型同样脆弱。 Despite explicit pixel-level operations, the five we tested (DeepEyes, DeepEyesV2, PixelReasoner, Mini-o3, CoF) sit at Accp ≈ 7–11 and CR ≈ 42–53. We do not claim pixel-tool methods cannot help in general — only that none of the five closed the gap on VisualFLIP. 尽管有显式的像素级操作，我们测试的 5 个模型（DeepEyes、DeepEyesV2、PixelReasoner、Mini-o3、CoF）均处于 Accp≈7–11、CR≈42–53。这并不意味着像素工具方法在一般情境下无效——只是这 5 个在 VisualFLIP 上都未能缩小差距。
  3. Sequential exposure amplifies prior-answer persistence.序列暴露放大了对先前答案的固守。 When the edited image follows an earlier answer in a sequential setting, several capable systems show lower Accp and higher collapse than under independent evaluation (see paper Table 2). 在序列设置下，当编辑图紧跟在先前答案之后出现，多个能力较强的模型相比独立评测呈现更低 Accp、更高坍塌（见论文 Table 2）。
07 / Use it使用

Three commands.三条命令。

pip install -U huggingface_hub requests
git clone https://github.com/didizhu-judy/VisualFLIP && cd VisualFLIP

# 1) Pull the dataset (~530 MB)
python scripts/download_data.py --out ./visualflip_data

# 2) Run an OpenRouter eval (auto-bumps max-tokens for the two long templates)
export VISUALFLIP_DATA=./visualflip_data
export OPENROUTER_API_KEY=sk-or-...
python scripts/evaluate.py --model google/gemini-2.5-flash \
    --out results/gemini25flash.json

# 3) Acc_p / CR / U-I-O decomposition + schema validation
python scripts/aggregate.py results/gemini25flash.json --expected-n 687

Weak models can skip the two long-context templates with --exclude-templates color_connectivity,maze_path. To submit a model to the leaderboard, open a PR — see CONTRIBUTING.md. 弱模型可加 --exclude-templates color_connectivity,maze_path 跳过两个长上下文模板。提交新模型到排行榜：发 PR，参见 CONTRIBUTING.md。
